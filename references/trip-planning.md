# 行程规划方法论

## 用户输入

### 最少要有

- 出发和回程日期、到达/离开时间
- 机场和航站楼
- 酒店名和地址
- 想去的点
- 不想去的类型

### 最好再给

- 人数、年龄段、有无长辈小孩
- 吃饭预算、能否排队、能走多少路
- 喜好：二次元/咖啡/酒吧/甜点/夜景/温泉/拍照/购物
- 行李多不多、交通偏好、愿不愿意早起

### 输入模板

```
到达：

酒店：
酒店地址：
人数：
年龄段：
一定要去：

```

## 判断顺序

1. 天数够不够
2. 酒店在哪
3. 用户讨厌什么
4. 哪些点需要预约
5. 哪些点会在节假日爆炸
6. 最后才补餐厅和咖啡

## 选点原则

- 一天一个主区域
- 一天只放一个重预约点
- 第一天下轻手
- 最后一天不跑远
- 长距离点只放在非节假日
- 用户明确讨厌的类型直接避开

## 选餐原则

- 先保路线顺
- 再保吃得好
- 再保空间和气氛
- 最后才考虑名气

## 执行步骤

### 1. 抽硬约束

到达时间、回程时间、航站楼 → 决定：
- 第一天不能跑远
- 最后一天不能排太满
- 每天按区域走

### 2. 用户清单分组

- 城内轻松组
- 预约组
- 远郊组
- 可路过组

### 3. 删高风险点

不是用户写了就全塞。主动删：
- 天数不够的远郊
- 节假日核心段会爆的热门远郊
- 看天气的景点（放不进短行程）

**明说删了什么，为什么删。**

### 4. 按区域重组

一天一个主区域 → 地铁换乘少、吃饭好插、人累得慢、临时改也容易。

### 5. 补吃饭

按当天路线近 → 空间气质对 → 不用为一顿饭卡死半天 → 有主推也有备选 → 最后一顿放稳。

### 6. 补票务和交通

只查关键的：官方票务、营业时间、休馆日、机场到酒店、酒店到机场。

### 7. 写执行文档

先说结论 → 每天去哪 → 每顿吃什么 → 主推和备选 → 砍掉什么 → 为什么砍 → 提前订什么 → 最后一天怎么去机场。

## 输出模板

```
先说结论

每天怎么走
- Day 1
- Day 2
- Day 3

每天吃什么
- 午餐主推 / 备选
- 晚餐主推 / 备选

提前订什么
删掉什么
为什么这样排
```

## 信息源优先级

1. 官方景点官网、票务页、机场页、交通页
2. Tabelog、Google Maps、餐厅官网
3. 社媒种草、小红书、游记 — 只补感觉，不能当营业时间

## 常见坑

- 一股脑把用户清单全塞进去
- 不看节假日
- 不看机场航站楼
- 不看最后一天返程时间
- 不看酒店位置
- 用社媒内容当营业时间
- 为了名店把整天卡死
- 给出"什么都想要"的行程

## 核心原则

不端水。要替用户删东西。行程不是越满越好，是越顺越好。

## 反问模板（用户只给一句话时）

只问最值钱的，不要问一大串：

1. 你们一共几个人，能不能走路？
2. 这次更想吃、逛店、看展，还是拍照？
3. 有没有一定要去和一定不要去的点？
4. 吃饭预算大概在哪一档？
5. 愿不愿意早起和排队？
